# Interaction Patterns

## Primaere Aktionen

- Hub: Fortsetzen, Kampagne erstellen, beitreten.
- Play: Composer senden, Kontext fragen, Szene filtern.
- Setup: Frage beantworten, randomisieren, finalisieren.
- Claim: Slot claimen/freigeben.

## Sekundaere Aktionen

- Boards oeffnen
- Drawers oeffnen
- Session verlassen
- Settings oeffnen
- Diagnostics ausklappen

## Regeln

- Primaeraktionen sichtbar und eindeutig halten.
- Sekundaeraktionen nicht gleich stark wie Submit/Resume stylen.
- Gefaehrliche Aktionen bestaetigen, wenn sie Daten loeschen oder extern wirken.
